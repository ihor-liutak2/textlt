Designed for focused work

Everything useful, nothing ceremonial.

TextLT is built for people who want a fast editor, not a ritual. Open a file, click, type, search, transform text, and keep moving.

⌨

Modeless editing

No command mode, no insert mode, no memorized dance. TextLT behaves closer to a GUI editor while staying in the terminal.

🎨

Beautiful TUI themes

Dark palettes, readable contrast, modal borders, selections, cursor colors, and syntax colors tuned for long coding sessions.

🧩

Lua text processors

Create repeatable text transformations for translation, cleanup, formatting, OCR fixes, or custom publishing workflows.

🖱

Mouse-aware terminal UI

Click to move, drag to select, use scroll wheels, operate modals, and navigate panels without leaving the terminal.

🔎

Search, replace, and files

Focused modals for file opening, recent files, search in files, find/replace, git tools, remote files, and structured workflows.

🗣

Local voice and AI-ready tools

TextLT is growing into a local writing workbench with Piper voice support, model registries, prompt builders, and practical automation.

Why TextLT feels different

A personal editor that can become your own workshop.

Most terminal editors are designed around old keyboard cultures. TextLT is different: it keeps terminal speed, but adds discoverable menus, modals, mouse support, custom processors, and a local-first philosophy.

For developers

C++, PHP, JS/TS, Python, Java, SQL, YAML, JSON, Docker, HTML/CSS, XML, configs, and more.

For translators

Text cleanup, paragraph wrapping, custom transformations, and keyboard-friendly editing in one place.

For local-first users

Your files stay on your machine. Add remote providers only when you want them.

For customization

Themes, shortcuts, pinned processors, modal workflows, and user-created Lua tools.

Install v0.9.1

Choose your platform, run the installer.

Pick Windows, Linux, or macOS. The box now only downloads one installer file and runs it; the real installation logic lives inside the platform installer script.

Installer scripts
The site now exposes one installer file per platform: Linux, Windows, and macOS. The copy box only downloads and runs that file.
Linux first-class
MX Linux, Ubuntu, Debian, WSL, and terminal workflows are the natural home of TextLT.
Windows supported
The archive installer creates a wrapper so resources such as themes and text processors load correctly.
macOS installer
The macOS installer uses the matching release archive for the current CPU architecture when that asset is published.
Static registries

Built for local tools and downloadable indexes.

The docs folder also publishes static JSON registries used by TextLT features. They stay untouched and can be fetched directly by the editor or viewed in a browser.
